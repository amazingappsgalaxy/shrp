## Aspect Ratio Mapping
The UI uses short form; RunningHub requires the full combo label:
| UI Value | RunningHub Value |
|----------|-----------------|
| 16:9 | 16:9 (Wide) |
| 4:3  | 4:3 (Standard) |
| 3:2  | 3:2 (Photo) |
| 1:1  | 1:1 (Square) — default fallback |
| 2:3  | 2:3 (Portrait Photo) |
| 3:4  | 3:4 (Portrait Standard) |
| 9:16 | 9:16 (Portrait Wide) |

NOTE: 21:9 is NOT supported by this model. Only the 7 values above.

## Implementation Notes
- Text-to-image only — imageUrl is passed as '' (empty string) to createTask
- validateRequest() is skipped for soul-2 since there is no imageUrl requirement
- Aspect ratio default fallback: '1:1 (Square)' if value not in map
- This model runs through startTaskForModel() (async, cron-polled flow)
  NOT through the synchronous handleSoul2Request() which is legacy code
